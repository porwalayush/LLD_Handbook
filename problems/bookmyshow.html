LLD Problem Hard

Design BookMyShow (End-to-End)

The capstone movie-ticketing problem. Model a full platform — cities, cinemas, screens, shows, seats — then nail the hard part: locking seats so two users never book the same one, even under heavy concurrent load.

📋 Problem Statement

Design an online movie-ticketing platform like BookMyShow. Users in a city search for movies, pick a show (a movie playing on a specific screen at a specific time), choose their seats, and pay. The core challenge: when a user selects seats, those seats must be locked temporarily so nobody else can grab them — but the lock must expire if the user abandons the flow or payment fails. The same seat must never be sold twice across thousands of simultaneous users.

Step 1 — Gather Requirements

This is a broad system, so scope it deliberately. Call out the must-haves first, then the qualities (concurrency above all) that make the design hard.

✓ Functional

  • Search movies by city, then list shows for a movie.
  • A show = movie + screen + start time, with its own seat map.
  • Show seats by type (regular / premium) and status (available / locked / booked).
  • Select seats → lock them with a timeout → pay → confirm booking.
  • Pluggable pricing per seat type and pluggable payment method.
  • Notify the user on booking confirmation.

◆ Non-Functional

  • Concurrency: the headline requirement — no double-booking of a seat, ever.
  • Lock expiry: abandoned selections free up automatically (TTL).
  • Extensibility: add pricing rules, payment methods, notification channels without edits.
  • Fast search across many cities and cinemas.
  • Consistency between lock state and booking state.

Step 2 — Identify the Core Entities

Pull the nouns out of the problem. A movie-ticketing platform has a clear geographic hierarchy down to the individual seat:

  • City — holds the cinemas a user can browse.
  • Cinema / Theater — a venue in a city; has many screens.
  • Screen / Audi — one auditorium; has a fixed physical seat layout.
  • Movie — title, language, duration.
  • Show — a movie on a screen at a time; owns the per-show seat states.
  • Seat & SeatType — a physical seat (row/number) and its class (REGULAR / PREMIUM).
  • ShowSeat — a seat within a show, with status AVAILABLE / LOCKED / BOOKED. This is the unit we lock.
  • Booking — a user's reservation of show seats; moves through a lifecycle (CREATED → CONFIRMED / EXPIRED / CANCELLED).
  • SeatLockProvider — grants short-lived locks with a TTL; the heart of concurrency control.
  • PricingStrategy / PaymentStrategy — pluggable pricing and payment.
  • BookingService — the Facade that orchestrates the whole search → lock → pay → confirm flow.

Step 4 — Patterns in Play

Naming the patterns as you design signals seniority. This end-to-end system pulls in five:

Singleton — one BookingService Facade — one entry point for the flow State — booking lifecycle Strategy — pricing & payment Observer — notifications Locking — safe seat hold

Step 5 — Implementation

The focus is the search → lock → book flow and the SeatLockProvider with TTL expiry. Everything funnels through the BookingService facade. This is a clean, runnable core — in an interview you'd grow it piece by piece.

enum SeatType { REGULAR, PREMIUM }
enum SeatStatus { AVAILABLE, LOCKED, BOOKED }
enum BookingStatus { CREATED, CONFIRMED, EXPIRED, CANCELLED }

class ShowSeat {
    final String id; final SeatType type;
    SeatStatus status = SeatStatus.AVAILABLE;
    ShowSeat(String id, SeatType type) { this.id = id; this.type = type; }
}

class Show {
    final String id; final String movie; final String screen;
    final Map<String, ShowSeat> seats = new HashMap<>();
    Show(String id, String movie, String screen) { this.id = id; this.movie = movie; this.screen = screen; }
}

// ── Concurrency core: short-lived locks keyed by (show, seat) ──
class SeatLock {
    final String user; final long expiresAt;
    SeatLock(String user, long expiresAt) { this.user = user; this.expiresAt = expiresAt; }
    boolean isExpired() { return System.currentTimeMillis() > expiresAt; }
}

class SeatLockProvider {
    private final long ttlMs;
    private final Map<String, SeatLock> locks = new HashMap<>();  // key = show:seat
    SeatLockProvider(long ttlMs) { this.ttlMs = ttlMs; }

    private String key(Show show, ShowSeat s) { return show.id + ":" + s.id; }

    // Atomically lock ALL requested seats, or none. synchronized = no race.
    synchronized boolean lock(Show show, List<ShowSeat> seats, String user) {
        for (ShowSeat s : seats) {                  // phase 1: check availability
            SeatLock existing = locks.get(key(show, s));
            boolean held = existing != null && !existing.isExpired();
            if (s.status == SeatStatus.BOOKED || held) return false;
        }
        long exp = System.currentTimeMillis() + ttlMs;
        for (ShowSeat s : seats) {                  // phase 2: commit all
            locks.put(key(show, s), new SeatLock(user, exp));
            s.status = SeatStatus.LOCKED;
        }
        return true;
    }

    synchronized void unlock(Show show, List<ShowSeat> seats) {
        for (ShowSeat s : seats) {
            locks.remove(key(show, s));
            if (s.status == SeatStatus.LOCKED) s.status = SeatStatus.AVAILABLE;
        }
    }

    synchronized boolean validate(Show show, List<ShowSeat> seats, String user) {
        for (ShowSeat s : seats) {
            SeatLock l = locks.get(key(show, s));
            if (l == null || l.isExpired() || !l.user.equals(user)) return false;
        }
        return true;
    }
}

// Strategy: pricing varies by seat type without touching the service
interface PricingStrategy { double price(List<ShowSeat> seats); }
class SeatTypePricing implements PricingStrategy {
    public double price(List<ShowSeat> seats) {
        double total = 0;
        for (ShowSeat s : seats) total += (s.type == SeatType.PREMIUM ? 400 : 250);
        return total;
    }
}

// Strategy: payment method is pluggable
interface PaymentStrategy { boolean pay(double amount); }
class CardPayment implements PaymentStrategy {
    public boolean pay(double amount) { return true; /* call gateway */ }
}

// Observer: notify on confirmation
interface BookingObserver { void onConfirmed(Booking b); }
class EmailNotifier implements BookingObserver {
    public void onConfirmed(Booking b) { System.out.println("Email: booking " + b.id + " confirmed"); }
}

class Booking {
    final String id; final Show show; final List<ShowSeat> seats;
    final String user; final double amount;
    BookingStatus status = BookingStatus.CREATED;
    Booking(String id, Show show, List<ShowSeat> seats, String user, double amt) {
        this.id = id; this.show = show; this.seats = seats; this.user = user; this.amount = amt;
    }
}

// Facade + Singleton: one orchestrator for the whole flow
class BookingService {
    private static final BookingService INSTANCE = new BookingService();
    public static BookingService get() { return INSTANCE; }

    private final SeatLockProvider lockProvider = new SeatLockProvider(300_000); // 5-min TTL
    private final PricingStrategy pricing = new SeatTypePricing();
    private final List<BookingObserver> observers = List.of(new EmailNotifier());
    private final Map<String, List<Show>> showsByMovie = new HashMap<>();

    // 1) SEARCH — list shows for a movie in a city
    public List<Show> search(String movie) {
        return showsByMovie.getOrDefault(movie, List.of());
    }

    // 2) LOCK — hold the selected seats for this user (TTL starts now)
    public boolean selectSeats(Show show, List<ShowSeat> seats, String user) {
        return lockProvider.lock(show, seats, user);
    }

    // 3) BOOK — validate lock, charge, confirm, notify, release lock
    public Booking book(Show show, List<ShowSeat> seats, String user, PaymentStrategy payment) {
        if (!lockProvider.validate(show, seats, user))
            throw new IllegalStateException("Lock expired — please reselect seats");

        double amount = pricing.price(seats);
        Booking b = new Booking(UUID.randomUUID().toString(), show, seats, user, amount);

        if (!payment.pay(amount)) {                  // payment failed → free the seats
            b.status = BookingStatus.EXPIRED;
            lockProvider.unlock(show, seats);
            throw new IllegalStateException("Payment failed");
        }

        for (ShowSeat s : seats) s.status = SeatStatus.BOOKED;  // commit the sale
        lockProvider.unlock(show, seats);            // lock done its job; clear it
        b.status = BookingStatus.CONFIRMED;
        for (BookingObserver o : observers) o.onConfirmed(b);
        return b;
    }
}

from enum import Enum
import time, threading, uuid

class SeatType(Enum): REGULAR = 1; PREMIUM = 2
class SeatStatus(Enum): AVAILABLE = 1; LOCKED = 2; BOOKED = 3
class BookingStatus(Enum): CREATED = 1; CONFIRMED = 2; EXPIRED = 3; CANCELLED = 4

class ShowSeat:
    def __init__(self, id, seat_type):
        self.id, self.type, self.status = id, seat_type, SeatStatus.AVAILABLE

class Show:
    def __init__(self, id, movie, screen):
        self.id, self.movie, self.screen = id, movie, screen
        self.seats = {}  # seat_id -> ShowSeat

# ── Concurrency core: short-lived locks keyed by (show, seat) ──
class SeatLock:
    def __init__(self, user, expires_at):
        self.user, self.expires_at = user, expires_at
    def is_expired(self): return time.time() > self.expires_at

class SeatLockProvider:
    def __init__(self, ttl_s):
        self.ttl_s, self.locks, self.lock = ttl_s, {}, threading.Lock()

    def _key(self, show, seat): return f"{show.id}:{seat.id}"

    # Lock ALL requested seats atomically, or none.
    def acquire(self, show, seats, user):
        with self.lock:                         # no race between check and commit
            for s in seats:                       # phase 1: check
                held = self.locks.get(self._key(show, s))
                if s.status == SeatStatus.BOOKED or (held and not held.is_expired()):
                    return False
            exp = time.time() + self.ttl_s
            for s in seats:                       # phase 2: commit
                self.locks[self._key(show, s)] = SeatLock(user, exp)
                s.status = SeatStatus.LOCKED
            return True

    def release(self, show, seats):
        with self.lock:
            for s in seats:
                self.locks.pop(self._key(show, s), None)
                if s.status == SeatStatus.LOCKED: s.status = SeatStatus.AVAILABLE

    def validate(self, show, seats, user):
        with self.lock:
            for s in seats:
                l = self.locks.get(self._key(show, s))
                if l is None or l.is_expired() or l.user != user:
                    return False
            return True

# Strategy: pricing by seat type
class SeatTypePricing:
    def price(self, seats):
        return sum(400 if s.type == SeatType.PREMIUM else 250 for s in seats)

# Strategy: pluggable payment
class CardPayment:
    def pay(self, amount): return True  # call gateway

# Observer: notify on confirmation
class EmailNotifier:
    def on_confirmed(self, booking):
        print(f"Email: booking {booking.id} confirmed")

class Booking:
    def __init__(self, id, show, seats, user, amount):
        self.id, self.show, self.seats = id, show, seats
        self.user, self.amount = user, amount
        self.status = BookingStatus.CREATED

# Facade + Singleton: one orchestrator for the whole flow
class BookingService:
    _instance = None
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance.lock_provider = SeatLockProvider(300)  # 5-min TTL
            cls._instance.pricing = SeatTypePricing()
            cls._instance.observers = [EmailNotifier()]
            cls._instance.shows_by_movie = {}
        return cls._instance

    # 1) SEARCH
    def search(self, movie):
        return self.shows_by_movie.get(movie, [])

    # 2) LOCK
    def select_seats(self, show, seats, user):
        return self.lock_provider.acquire(show, seats, user)

    # 3) BOOK — validate, charge, confirm, notify, release
    def book(self, show, seats, user, payment):
        if not self.lock_provider.validate(show, seats, user):
            raise RuntimeError("Lock expired — please reselect seats")

        amount = self.pricing.price(seats)
        b = Booking(str(uuid.uuid4()), show, seats, user, amount)

        if not payment.pay(amount):                # payment failed → free seats
            b.status = BookingStatus.EXPIRED
            self.lock_provider.release(show, seats)
            raise RuntimeError("Payment failed")

        for s in seats: s.status = SeatStatus.BOOKED  # commit the sale
        self.lock_provider.release(show, seats)        # lock done; clear it
        b.status = BookingStatus.CONFIRMED
        for o in self.observers: o.on_confirmed(b)
        return b
Why this design scores well: the SeatLockProvider isolates the one truly hard part — concurrency — behind a tiny interface, and its lock/commit happens in two phases under a single lock so a multi-seat selection is all-or-nothing. The TTL means abandoned carts and failed payments self-heal without a cleanup job racing the user. The BookingService facade hides the lock → price → pay → confirm → notify sequence behind three clean methods, while PricingStrategy, PaymentStrategy, and BookingObserver let you add a weekend surcharge, a new wallet, or an SMS channel as a new class — never an edit (Open/Closed).

Step 6 — Follow-up Extensions

This is the capstone problem, so interviewers push hard on scale. Be ready for:

  • Distributed seat locking: the in-memory synchronized map only works on one node. Replace it with Redis SET key value NX PX ttl — atomic lock-if-absent with built-in expiry — so locks are shared across a fleet of servers.
  • Waiting queues: for blockbuster releases, put users in a virtual queue (token + position) before they can even reach seat selection, smoothing the thundering herd.
  • Dynamic pricing: a new PricingStrategy that factors in demand, time-to-showtime, and occupancy — surge pricing without touching the booking flow.
  • Scaling search across cities: shard show/seat data by city or cinema; back search with Elasticsearch and a CDN-cached movie catalog.
  • Idempotent payments: use a payment idempotency key so a retried request never double-charges or double-books.
  • Persistence & audit: store bookings in a database with the seat-booked transaction as the source of truth; the in-memory state becomes a cache.
Common mistakes: putting the seat status flag only on the seat and skipping the lock provider — so two threads both read AVAILABLE and both book; locking seats one-at-a-time instead of all-or-nothing (you end up holding 2 of 4 seats and deadlocking with another user); forgetting the TTL so abandoned carts freeze seats forever; confirming the booking before payment succeeds; and modeling status as free-form strings instead of enums. Guard the lock, expire it, and confirm only after payment — that is the whole problem.

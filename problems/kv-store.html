LLD Problem Hard

Design an In-Memory Key-Value Store

A Redis-style store supporting O(1) get/put/delete, per-key TTL expiry, a pluggable eviction policy (LRU/LFU), and transactions. The interview centerpiece is a correct, O(1) LRU cache built from a HashMap plus a doubly-linked list.

📋 Problem Statement

Design an in-memory KeyValueStore with get(k), put(k, v), and delete(k). The store has a fixed capacity; when full, it evicts an entry chosen by a pluggable EvictionPolicy (LRU or LFU). Keys may have a TTL and expire lazily. The store must support simple transactions — begin / commit / rollback — and be safe under concurrent access. All core operations should be O(1).

Step 1 — Gather Requirements

Scope first. The hard part isn't the API — it's hitting O(1) for every operation including eviction.

✓ Functional

  • get, put, delete on string keys.
  • Fixed capacity with automatic eviction when full.
  • Pluggable eviction policy: LRU or LFU.
  • Optional per-key TTL; expired keys disappear.
  • Transactions: begin / commit / rollback.

◆ Non-Functional

  • O(1) get / put / delete / evict.
  • Thread-safe under concurrent readers and writers.
  • Extensibility: add eviction policies without editing the store.
  • Bounded memory — never exceed capacity.

Step 2 — Identify the Core Entities

The nouns become classes:

  • KeyValueStore — the facade: get/put/delete, holds the map and capacity.
  • Entry / Node — value + expiry timestamp; in the LRU it's a doubly-linked-list node.
  • EvictionPolicy (interface) — Strategy: recordAccess, recordInsert, evict.
  • LRUPolicy / LFUPolicy — concrete strategies; LRU uses a HashMap + doubly-linked list.
  • TransactionManager — stacks change-sets (Memento/Command) so rollback restores prior state.

Step 3 — Class Diagram

Step 4 — Patterns in Play

Name the patterns — they map cleanly onto this problem:

Strategy — pluggable eviction (LRU/LFU) Memento — transaction rollback log Command — replayable write ops Locking — thread-safe access

Step 5 — Implementation

The heart of the answer: a fully working O(1) LRU cache — HashMap for lookup, doubly-linked list (with sentinel head/tail) for recency. TTL is checked lazily on read; transactions are sketched at the end.

import java.util.*;

// Doubly-linked-list node: also the stored Entry (value + expiry)
class Node {
    String key, value; long expireAt;   // 0 = never expires
    Node prev, next;
    Node(String k, String v, long exp) { key = k; value = v; expireAt = exp; }
}

// ---- Strategy: an LRU eviction policy backed by a doubly-linked list ----
// head <-> (most recent) ... (least recent) <-> tail   (both are sentinels)
class LRUStore {
    private final int capacity;
    private final Map<String, Node> map = new HashMap<>();
    private final Node head = new Node(null, null, 0);
    private final Node tail = new Node(null, null, 0);
    private final Object lock = new Object();   // thread-safety guard

    LRUStore(int capacity) {
        this.capacity = capacity;
        head.next = tail; tail.prev = head;     // empty list
    }

    // --- O(1) list surgery helpers ---
    private void remove(Node n) { n.prev.next = n.next; n.next.prev = n.prev; }
    private void addFront(Node n) {          // insert right after head = most recent
        n.next = head.next; n.prev = head;
        head.next.prev = n; head.next = n;
    }
    private void touch(Node n) { remove(n); addFront(n); }   // mark as recently used

    private boolean expired(Node n) {
        return n.expireAt != 0 && System.currentTimeMillis() > n.expireAt;
    }

    String get(String key) {
        synchronized (lock) {
            Node n = map.get(key);
            if (n == null) return null;
            if (expired(n)) { remove(n); map.remove(key); return null; }  // lazy TTL
            touch(n);                          // most-recently-used now
            return n.value;
        }
    }

    void put(String key, String value, long ttlMs) {
        synchronized (lock) {
            long exp = ttlMs > 0 ? System.currentTimeMillis() + ttlMs : 0;
            Node n = map.get(key);
            if (n != null) {                  // update in place + promote
                n.value = value; n.expireAt = exp; touch(n); return;
            }
            if (map.size() >= capacity) {       // evict LRU = node before tail
                Node lru = tail.prev;
                remove(lru); map.remove(lru.key);
            }
            Node fresh = new Node(key, value, exp);
            addFront(fresh); map.put(key, fresh);
        }
    }

    void delete(String key) {
        synchronized (lock) {
            Node n = map.remove(key);
            if (n != null) remove(n);
        }
    }

    public static void main(String[] a) {
        LRUStore kv = new LRUStore(2);
        kv.put("a", "1", 0);
        kv.put("b", "2", 0);
        kv.get("a");                // 'a' is now most-recent
        kv.put("c", "3", 0);       // capacity hit -> evicts 'b' (LRU)
        System.out.println(kv.get("b"));  // null
        System.out.println(kv.get("a"));  // 1
        System.out.println(kv.get("c"));  // 3
    }
}

// ---- Memento: transactions stack undo logs; rollback replays them ----
// begin()  -> push an empty change-set onto a stack
// on each put/delete inside a txn, record the OLD value in the top set
// commit() -> discard the change-set (changes already applied)
// rollback() -> pop the set and restore every recorded OLD value

import time, threading

class Node:                       # DLL node = stored Entry (value + expiry)
    def __init__(self, key, value, expire_at=0):   # 0 = never expires
        self.key, self.value, self.expire_at = key, value, expire_at
        self.prev = self.next = None

# ---- O(1) LRU: HashMap for lookup + doubly-linked list for recency ----
# head <-> (most recent) ... (least recent) <-> tail   (both sentinels)
class LRUStore:
    def __init__(self, capacity):
        self.capacity = capacity
        self.map = {}
        self.head, self.tail = Node(None, None), Node(None, None)
        self.head.next, self.tail.prev = self.tail, self.head
        self.lock = threading.RLock()       # thread-safety guard

    # --- O(1) list surgery helpers ---
    def _remove(self, n):
        n.prev.next, n.next.prev = n.next, n.prev
    def _add_front(self, n):              # right after head = most recent
        n.next, n.prev = self.head.next, self.head
        self.head.next.prev = n
        self.head.next = n
    def _touch(self, n):                  # mark recently used
        self._remove(n); self._add_front(n)

    def _expired(self, n):
        return n.expire_at and time.time() > n.expire_at

    def get(self, key):
        with self.lock:
            n = self.map.get(key)
            if n is None: return None
            if self._expired(n):           # lazy TTL
                self._remove(n); del self.map[key]; return None
            self._touch(n)                  # most-recently-used now
            return n.value

    def put(self, key, value, ttl=0):
        with self.lock:
            exp = time.time() + ttl if ttl > 0 else 0
            n = self.map.get(key)
            if n:                          # update in place + promote
                n.value, n.expire_at = value, exp
                self._touch(n); return
            if len(self.map) >= self.capacity:   # evict LRU = node before tail
                lru = self.tail.prev
                self._remove(lru); del self.map[lru.key]
            fresh = Node(key, value, exp)
            self._add_front(fresh)
            self.map[key] = fresh

    def delete(self, key):
        with self.lock:
            n = self.map.pop(key, None)
            if n: self._remove(n)

if __name__ == "__main__":
    kv = LRUStore(2)
    kv.put("a", "1")
    kv.put("b", "2")
    kv.get("a")                # 'a' is now most-recent
    kv.put("c", "3")           # capacity hit -> evicts 'b' (LRU)
    print(kv.get("b"))         # None
    print(kv.get("a"))         # 1
    print(kv.get("c"))         # 3

# ---- Memento: transactions as a stack of undo logs ----
# begin()    -> self._txn.append({})            push a fresh change-set
# put/delete -> if in txn, record OLD value once in the top change-set
# commit()   -> self._txn.pop()                  changes already applied
# rollback() -> for k, old in self._txn.pop().items(): restore old value
Why this design scores well: the LRU hits true O(1) for every operation because the HashMap gives O(1) lookup and the doubly-linked list gives O(1) move-to-front and O(1) eviction (the node just before tail). The sentinel head/tail nodes eliminate every null-check during list surgery — a detail interviewers love. TTL is checked lazily on read (no background thread needed for the core), and the EvictionPolicy is a Strategy so swapping to LFU never touches get/put. Rollback is a Memento: an undo log of prior values.

Step 6 — Follow-up Extensions

Where the interviewer pushes next:

  • LFU eviction: add frequency buckets (a map of freq → ordered list) for O(1) least-frequent eviction; same EvictionPolicy interface.
  • Active TTL expiry: a background sweeper or a min-heap of expirations to reclaim memory eagerly, not just on read.
  • Concurrency at scale: shard the store into N segments each with its own lock (like ConcurrentHashMap) to cut contention — see locks & synchronization.
  • Nested transactions: the change-set stack already supports nesting; rollback pops one level, commit merges into the parent.
  • Persistence: append-only write log (like Redis AOF) replayed on restart; snapshots for fast recovery.
Common mistakes: using only a HashMap and a list — eviction becomes O(n) because you scan for the LRU; forgetting sentinel head/tail and drowning in null-checks; checking TTL only in get but never reclaiming the memory; hardcoding LRU instead of a Strategy so LFU forces a rewrite; and omitting the lock so concurrent puts corrupt the linked list. Avoid all five and you're in the top tier.

LLD Problem Easy

Design a Logging Framework

The textbook Chain of Responsibility problem. A log message flows through a chain of level-based handlers — each either handles it or passes it along — so a single log() call routes to the right destination by severity.

📋 Problem Statement

Design a logging framework like Log4j. Application code calls logger.info("...") or logger.error("..."). Each log level (DEBUG, INFO, WARN, ERROR) should be processed by an appropriate handler, and a message can be written to one or more destinations — console, file, etc. The design must make it easy to add new levels and destinations, and be safe to call from many threads.

Step 1 — Gather Requirements

Start by pinning down scope. Say your assumptions out loud — interviewers want to see you separate the must-haves from the nice-to-haves.

✓ Functional

  • Four levels: DEBUG < INFO < WARN < ERROR.
  • A single log(level, message) entry point.
  • Each level handled by a dedicated handler; lower-priority handlers pass up the chain.
  • Pluggable destinations: console, file (appenders).
  • Configurable minimum level (drop everything below it).

◆ Non-Functional

  • Thread-safety: many threads log concurrently.
  • Extensibility: add a level or appender without editing existing code.
  • Single global logger (Singleton) shared across the app.
  • Low overhead when a message is below the threshold.

Step 2 — Identify the Core Entities

Find the nouns in the problem. These become your classes:

  • Logger — the public entry point (often a Singleton); kicks off the chain.
  • LogLevel — ordered enum DEBUG/INFO/WARN/ERROR.
  • LogMessage — bundles the level, text, and timestamp.
  • LogHandler (abstract) — a link in the chain; has a level and a next pointer.
  • DebugHandler, InfoHandler, WarnHandler, ErrorHandler — concrete links.
  • Appender — where text actually goes: ConsoleAppender, FileAppender.
  • Formatter — turns a LogMessage into a string.

Step 3 — Class Diagram

The key idea: a message enters the chain at the lowest level and walks up. A handler processes it if its own level matches, then forwards to next regardless — so an ERROR can be handled by ERROR while DEBUG simply ignores it.

Logger (Singleton) ──starts──▶ chain head
   │ log(level, msg)
   ▼
LogHandler (abstract) { level · next · handle(msg) · write(msg) }
   ▲ next ──▶ next ──▶ next ──▶ next
   │
   ├── DebugHandler ─▶ InfoHandler ─▶ WarnHandler ─▶ ErrorHandler

LogHandler ──uses──▶ Appender (interface)
                          ▲
              ┌───────────┴───────────┐
        ConsoleAppender          FileAppender
Appender ──uses──▶ Formatter ──formats──▶ LogMessage {level, text, ts}

  DEBUG msg -> Debug handles -> passes up (others ignore)
  ERROR msg -> Debug/Info/Warn pass -> Error handles

Step 4 — Patterns in Play

Name the patterns as you design — it signals maturity. This problem is the canonical home of Chain of Responsibility:

Step 5 — Implementation

Each handler knows only its own level. handle() writes if the message's level is at least its own, then always forwards to next — that single rule, repeated down the chain, routes everything correctly.

enum LogLevel { DEBUG, INFO, WARN, ERROR }   // ordinal gives priority

class LogMessage {
    final LogLevel level; final String text; final long ts;
    LogMessage(LogLevel level, String text) {
        this.level = level; this.text = text; this.ts = System.currentTimeMillis();
    }
}

// Strategy: how a message becomes a line of text
interface Formatter { String format(LogMessage m); }
class SimpleFormatter implements Formatter {
    public String format(LogMessage m) { return m.ts + " [" + m.level + "] " + m.text; }
}

// Appender: where text physically goes
interface Appender { void append(String line); }
class ConsoleAppender implements Appender {
    public synchronized void append(String line) { System.out.println(line); }
}

// Chain of Responsibility: each handler owns one level
abstract class LogHandler {
    protected final LogLevel level;
    protected LogHandler next;
    protected final Appender appender;
    protected final Formatter formatter = new SimpleFormatter();

    LogHandler(LogLevel level, Appender appender) { this.level = level; this.appender = appender; }
    LogHandler setNext(LogHandler next) { this.next = next; return next; }

    void handle(LogMessage m) {
        if (m.level.ordinal() >= level.ordinal()) {   // this link cares
            appender.append(formatter.format(m));
        }
        if (next != null) next.handle(m);          // always pass it on
    }
}
class DebugHandler extends LogHandler { DebugHandler(Appender a) { super(LogLevel.DEBUG, a); } }
class InfoHandler  extends LogHandler { InfoHandler(Appender a)  { super(LogLevel.INFO, a); } }
class WarnHandler  extends LogHandler { WarnHandler(Appender a)  { super(LogLevel.WARN, a); } }
class ErrorHandler extends LogHandler { ErrorHandler(Appender a) { super(LogLevel.ERROR, a); } }

class Logger {                                  // Singleton entry point
    private static final Logger INSTANCE = new Logger();
    public static Logger get() { return INSTANCE; }

    private final LogHandler chain;
    private Logger() {
        Appender console = new ConsoleAppender();
        chain = new DebugHandler(console);    // build the chain head -> ... -> tail
        chain.setNext(new InfoHandler(console))
             .setNext(new WarnHandler(console))
             .setNext(new ErrorHandler(console));
    }
    public void log(LogLevel level, String text) { chain.handle(new LogMessage(level, text)); }
    public void debug(String t) { log(LogLevel.DEBUG, t); }
    public void info(String t)  { log(LogLevel.INFO, t); }
    public void warn(String t)  { log(LogLevel.WARN, t); }
    public void error(String t) { log(LogLevel.ERROR, t); }
}

// Usage:  Logger.get().error("disk full");  // walks chain, Error link writes it

from enum import IntEnum
from abc import ABC, abstractmethod
import time, threading

class LogLevel(IntEnum):              # value gives priority
    DEBUG = 0; INFO = 1; WARN = 2; ERROR = 3

class LogMessage:
    def __init__(self, level, text):
        self.level, self.text, self.ts = level, text, time.time()

# Strategy: how a message becomes a line of text
class SimpleFormatter:
    def format(self, m):
        return f"{m.ts:.0f} [{m.level.name}] {m.text}"

# Appender: where text physically goes
class Appender(ABC):
    @abstractmethod
    def append(self, line): ...

class ConsoleAppender(Appender):
    def __init__(self): self._lock = threading.Lock()
    def append(self, line):
        with self._lock:                 # thread-safe write
            print(line)

# Chain of Responsibility: each handler owns one level
class LogHandler:
    def __init__(self, level, appender):
        self.level, self.appender = level, appender
        self.next = None
        self.formatter = SimpleFormatter()
    def set_next(self, nxt):
        self.next = nxt; return nxt        # chainable
    def handle(self, m):
        if m.level >= self.level:         # this link cares
            self.appender.append(self.formatter.format(m))
        if self.next:                    # always pass it on
            self.next.handle(m)

class DebugHandler(LogHandler):
    def __init__(self, a): super().__init__(LogLevel.DEBUG, a)
class InfoHandler(LogHandler):
    def __init__(self, a): super().__init__(LogLevel.INFO, a)
class WarnHandler(LogHandler):
    def __init__(self, a): super().__init__(LogLevel.WARN, a)
class ErrorHandler(LogHandler):
    def __init__(self, a): super().__init__(LogLevel.ERROR, a)

class Logger:                          # Singleton entry point
    _instance = None
    _lock = threading.Lock()
    def __new__(cls):
        with cls._lock:                  # thread-safe singleton
            if cls._instance is None:
                cls._instance = super().__new__(cls)
                console = ConsoleAppender()
                head = DebugHandler(console)   # build the chain
                head.set_next(InfoHandler(console)) \
                    .set_next(WarnHandler(console)) \
                    .set_next(ErrorHandler(console))
                cls._instance.chain = head
        return cls._instance
    def log(self, level, text): self.chain.handle(LogMessage(level, text))
    def debug(self, t): self.log(LogLevel.DEBUG, t)
    def info(self, t):  self.log(LogLevel.INFO, t)
    def warn(self, t):  self.log(LogLevel.WARN, t)
    def error(self, t): self.log(LogLevel.ERROR, t)

# --- demo ---
log = Logger()
log.error("disk full")   # walks chain; Error link writes it
log.debug("cache hit")   # Debug link writes it; others ignore
Why this design scores well: the Logger never branches on level with if/else — it just hands the message to the chain head. Each handler applies one tiny rule (handle-if-mine, always-forward), so routing emerges from the chain's structure. Swapping a ConsoleAppender for a FileAppender, or adding a FATAL level, touches no existing handler. The synchronized/Lock on the appender is the thread-safety detail interviewers probe.

Step 6 — Follow-up Extensions

Strong candidates anticipate where the interviewer pushes next:

  • Multiple appenders per level: give a handler a list of appenders (console + file + network).
  • Async logging: push messages onto a queue and drain on a background thread — a Producer/Consumer setup.
  • Per-package loggers: a hierarchy where child loggers inherit the parent's level/appenders.
  • Rolling files: a RollingFileAppender that rotates by size or date.
  • Structured logs: a JSON Formatter via the Strategy slot.
Common mistakes: writing one method with a giant switch(level) (the anti-pattern Chain of Responsibility removes); forgetting to forward to next so messages get dropped; not synchronizing the appender so concurrent lines interleave; and hardcoding System.out instead of an Appender abstraction. Model handlers as a chain and destinations as appenders, and the framework stays open for extension.

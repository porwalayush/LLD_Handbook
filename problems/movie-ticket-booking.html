Design Movie Ticket Booking

The quintessential concurrency LLD problem. Two users tap the same seat at the same instant — your design must guarantee only one of them gets it. The answer is seat locking with a timeout.

📋 Problem Statement

Design the seat-booking core of a movie ticketing system. A theater has screens; each screen runs shows of a movie at given times. A show has a grid of seats, each AVAILABLE, LOCKED, or BOOKED. A user selects seats, the system locks them for a short window, the user pays, and on success the seats are confirmed into a booking. The hard part: many users booking the same seat concurrently must never double-book — and abandoned locks must auto-expire.

Step 1 — Gather Requirements

Always start by pinning down scope. State your assumptions out loud; interviewers want to see you separate the must-haves from the nice-to-haves.

✓ Functional

  • Browse movies, theaters, and shows.
  • View seat map for a show with live status.
  • Lock selected seats temporarily on selection.
  • Pay within a timeout to confirm the booking.
  • Auto-release seats if payment doesn't happen in time.
  • Cancel a booking and free the seats.

◆ Non-Functional

  • No double-booking under heavy concurrency.
  • Lock timeout so abandoned carts free seats.
  • Low contention: lock per show, not globally.
  • Consistency: a seat is in exactly one state.
  • Fast seat-map reads for popular shows.

Step 2 — Identify the Core Entities

Find the nouns in the problem. These become your classes:

  • Movie — title, duration, language.
  • Theater — a venue with multiple screens.
  • Screen — a physical auditorium; hosts shows.
  • Show — a movie on a screen at a time; owns the seat map.
  • Seat — a position with a status: AVAILABLE / LOCKED / BOOKED.
  • Booking — confirmed seats for a user and show.
  • Payment — charges the user; success drives confirmation.
  • User — the person booking.
  • SeatLockManager — the concurrency heart: locks seats with a timeout.

Step 5 — Implementation

The whole problem lives in the critical section: locking a seat must be atomic. Below, a per-show lock guards a check-then-act so two threads cannot both see a seat as AVAILABLE and grab it. Locks carry an expiry so abandoned selections free up.

enum SeatStatus { AVAILABLE, LOCKED, BOOKED }

class Seat {
    final String id;
    volatile SeatStatus status = SeatStatus.AVAILABLE;
    String lockedBy;            // user holding the lock
    long lockExpiresAt;          // epoch ms; 0 = not locked
    Seat(String id) { this.id = id; }
}

class Show {
    final String id;
    final Map<String, Seat> seats = new ConcurrentHashMap<>();
    // One lock per show keeps contention low (not one global lock)
    final ReentrantLock lock = new ReentrantLock();
    Show(String id) { this.id = id; }
}

class SeatLockManager {
    private final long lockTtlMs;
    SeatLockManager(long lockTtlMs) { this.lockTtlMs = lockTtlMs; }

    // THE critical section: atomically lock all requested seats.
    // All-or-nothing — if any seat is taken, lock none.
    boolean lockSeats(Show show, List<String> seatIds, String userId) {
        show.lock.lock();                       // enter critical section
        try {
            long now = System.currentTimeMillis();
            for (String id : seatIds) {           // check phase
                Seat s = show.seats.get(id);
                if (!isFree(s, now)) return false;     // someone else has it
            }
            for (String id : seatIds) {           // act phase
                Seat s = show.seats.get(id);
                s.status = SeatStatus.LOCKED;
                s.lockedBy = userId;
                s.lockExpiresAt = now + lockTtlMs;
            }
            return true;
        } finally {
            show.lock.unlock();                   // always release
        }
    }

    // A seat is free if AVAILABLE, or its lock has expired
    private boolean isFree(Seat s, long now) {
        if (s.status == SeatStatus.BOOKED) return false;
        if (s.status == SeatStatus.LOCKED && now < s.lockExpiresAt) return false;
        return true;                          // AVAILABLE or stale lock
    }

    // Confirm only the seats this user actually locked
    boolean confirm(Show show, List<String> seatIds, String userId) {
        show.lock.lock();
        try {
            long now = System.currentTimeMillis();
            for (String id : seatIds) {
                Seat s = show.seats.get(id);
                if (s.status != SeatStatus.LOCKED
                        || !userId.equals(s.lockedBy)
                        || now >= s.lockExpiresAt) return false;  // lock lost/expired
            }
            for (String id : seatIds)
                show.seats.get(id).status = SeatStatus.BOOKED;
            return true;
        } finally { show.lock.unlock(); }
    }
}

class BookingService {
    final SeatLockManager locks = new SeatLockManager(300_000); // 5-min hold
    final PaymentGateway gateway;
    BookingService(PaymentGateway g) { gateway = g; }

    // select → lock → pay → confirm
    Booking book(Show show, List<String> seatIds, String userId) {
        if (!locks.lockSeats(show, seatIds, userId))
            throw new IllegalStateException("Seats no longer available");
        if (!gateway.charge(userId, seatIds.size() * 250))
            throw new IllegalStateException("Payment failed");   // lock expires on its own
        if (!locks.confirm(show, seatIds, userId))
            throw new IllegalStateException("Lock expired before confirm");
        return new Booking(userId, show.id, seatIds);
    }
}

import threading, time
from enum import Enum

class SeatStatus(Enum):
    AVAILABLE = 1; LOCKED = 2; BOOKED = 3

class Seat:
    def __init__(self, id):
        self.id = id
        self.status = SeatStatus.AVAILABLE
        self.locked_by = None
        self.lock_expires_at = 0.0      # epoch seconds

class Show:
    def __init__(self, id):
        self.id = id
        self.seats = {}                  # seat_id -> Seat
        self.lock = threading.Lock()    # one lock per show

class SeatLockManager:
    def __init__(self, ttl_s):
        self.ttl_s = ttl_s

    # THE critical section: lock all seats atomically (all-or-nothing)
    def lock_seats(self, show, seat_ids, user_id):
        with show.lock:                  # enter critical section
            now = time.time()
            for sid in seat_ids:           # check phase
                if not self._is_free(show.seats[sid], now):
                    return False           # someone else has it
            for sid in seat_ids:           # act phase
                s = show.seats[sid]
                s.status = SeatStatus.LOCKED
                s.locked_by = user_id
                s.lock_expires_at = now + self.ttl_s
            return True

    def _is_free(self, s, now):
        if s.status == SeatStatus.BOOKED:
            return False
        if s.status == SeatStatus.LOCKED and now < s.lock_expires_at:
            return False
        return True                     # AVAILABLE or stale lock

    def confirm(self, show, seat_ids, user_id):
        with show.lock:
            now = time.time()
            for sid in seat_ids:
                s = show.seats[sid]
                if (s.status != SeatStatus.LOCKED
                        or s.locked_by != user_id
                        or now >= s.lock_expires_at):
                    return False           # lock lost/expired
            for sid in seat_ids:
                show.seats[sid].status = SeatStatus.BOOKED
            return True

class BookingService:
    def __init__(self, gateway):
        self.locks = SeatLockManager(ttl_s=300)   # 5-min hold
        self.gateway = gateway

    # select → lock → pay → confirm
    def book(self, show, seat_ids, user_id):
        if not self.locks.lock_seats(show, seat_ids, user_id):
            raise RuntimeError("Seats no longer available")
        if not self.gateway.charge(user_id, len(seat_ids) * 250):
            raise RuntimeError("Payment failed")     # lock expires on its own
        if not self.locks.confirm(show, seat_ids, user_id):
            raise RuntimeError("Lock expired before confirm")
        return {"user": user_id, "show": show.id, "seats": seat_ids}
Why this design scores well: the entire correctness argument rests on the critical section. By holding show.lock while doing the check-then-act (verify every seat is free, then mark them LOCKED), two concurrent book() calls for the same seat are serialized — the second one finds the seat LOCKED and bails. The lock is held per show, not globally, so unrelated shows never contend. And the lockExpiresAt timeout means a user who walks away after locking seats doesn't freeze them forever — isFree treats a stale lock as available.

Step 6 — Follow-up Extensions

Strong candidates anticipate where the interviewer pushes next:

  • Distributed locking: with many app servers, an in-process ReentrantLock isn't enough. Move to a shared store — Redis SET NX PX with the user as the value and a TTL, or a DB row with optimistic UPDATE ... WHERE status='AVAILABLE' (compare-and-swap).
  • DB-level guarantee: a unique constraint on (show_id, seat_id) in a bookings table makes double-booking impossible even if app logic slips.
  • Lock reaper: a background sweeper actively resets expired LOCKED seats to AVAILABLE for accurate seat-map reads (instead of only lazy expiry).
  • Idempotent payments: use a payment token so a retried charge doesn't double-bill.
  • Seat-map performance: cache the read-mostly map and invalidate on state change.
Common mistakes: doing the seat check and the seat update outside one lock, leaving a check-then-act race that double-books; using one global lock so the whole site serializes on a single popular show; locking seats with no timeout so abandoned carts freeze inventory forever; confirming seats without verifying the caller actually holds the lock; and forgetting to release the lock in a finally block so an exception strands it.

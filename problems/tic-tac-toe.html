LLD Problem Easy

Design Tic-Tac-Toe

A perfect warm-up LLD problem. Model a two-player NxN grid game with clean turn handling and correct win/draw detection — then make the win check fast.

📋 Problem Statement

Design a Tic-Tac-Toe game for two players on an N×N board (classic is 3×3). Players alternate placing their symbol (X or O) in an empty cell. The first player to fill an entire row, column, or diagonal with their symbol wins. If the board fills with no winner, the game is a draw. The system must reject moves on occupied or out-of-bounds cells.

Step 1 — Gather Requirements

Even for a "simple" game, pin the scope down first. Calling out the NxN generalization and the win-check cost shows the interviewer you think beyond the toy case.

✓ Functional

  • Configurable N×N board (default 3×3).
  • Two players, each owning a symbol (X / O).
  • Place a symbol at (row, col); reject if occupied or off-board.
  • Alternate turns automatically.
  • Detect a win (row / column / either diagonal) and a draw.

◆ Non-Functional

  • Efficient win check: avoid rescanning the whole board every move.
  • Extensibility: easy to vary N or add players/symbols.
  • Clear state: a single source of truth for whose turn it is and game status.
  • Validation: illegal moves never corrupt state.

Step 2 — Identify the Core Entities

Pull the nouns out of the statement — they become your classes:

  • Game — orchestrates the match: holds players, the board, whose turn it is, and the status.
  • Board — the N×N grid of cells; exposes place() and win/draw checks.
  • Cell — one square; holds a Symbol or is empty (model as an enum value).
  • Player — a participant bound to one Symbol.
  • Symbol (enum) — X, O, or EMPTY — the cell state.
  • Move — a (player, row, col) action; useful for history/undo later.
  • GameStatus (enum) — IN_PROGRESS, WIN, DRAW.

Step 5 — Implementation

A clean, runnable core: a Symbol enum, a Board that validates and places, and a Game loop that alternates turns and checks for a result after each move.

enum Symbol { X, O, EMPTY }
enum GameStatus { IN_PROGRESS, WIN, DRAW }

class Player {
    final String name; final Symbol symbol;
    Player(String name, Symbol symbol) { this.name = name; this.symbol = symbol; }
}

class Board {
    final int n;
    private final Symbol[][] grid;
    private int filled = 0;

    Board(int n) {
        this.n = n;
        grid = new Symbol[n][n];
        for (Symbol[] row : grid) Arrays.fill(row, Symbol.EMPTY);
    }

    // places the symbol; throws on an illegal move
    void place(Symbol s, int r, int c) {
        if (r < 0 || r >= n || c < 0 || c >= n)
            throw new IllegalArgumentException("Off board");
        if (grid[r][c] != Symbol.EMPTY)
            throw new IllegalStateException("Cell taken");
        grid[r][c] = s;
        filled++;
    }

    boolean isFull() { return filled == n * n; }

    // only re-scan the row, column & diagonals touched by (r, c)
    boolean checkWinner(Symbol s, int r, int c) {
        boolean row = true, col = true, diag = (r == c), anti = (r + c == n - 1);
        for (int i = 0; i < n; i++) {
            if (grid[r][i] != s) row = false;
            if (grid[i][c] != s) col = false;
            if (diag && grid[i][i] != s) diag = false;
            if (anti && grid[i][n - 1 - i] != s) anti = false;
        }
        return row || col || diag || anti;
    }
}

class Game {
    private final Board board;
    private final Player[] players;
    private int turn = 0;                       // index into players
    private GameStatus status = GameStatus.IN_PROGRESS;

    Game(int n, Player p1, Player p2) {
        board = new Board(n);
        players = new Player[]{ p1, p2 };
    }

    // one turn: validate, place, then test for win/draw
    public GameStatus move(Player p, int r, int c) {
        if (status != GameStatus.IN_PROGRESS)
            throw new IllegalStateException("Game over");
        if (p != players[turn])
            throw new IllegalStateException("Not your turn");

        board.place(p.symbol, r, c);
        if (board.checkWinner(p.symbol, r, c)) status = GameStatus.WIN;
        else if (board.isFull())          status = GameStatus.DRAW;
        else                                turn = (turn + 1) % 2;  // alternate
        return status;
    }
}

from enum import Enum

class Symbol(Enum):
    X = "X"; O = "O"; EMPTY = "."

class GameStatus(Enum):
    IN_PROGRESS = 1; WIN = 2; DRAW = 3

class Player:
    def __init__(self, name, symbol):
        self.name, self.symbol = name, symbol

class Board:
    def __init__(self, n):
        self.n = n
        self.grid = [[Symbol.EMPTY] * n for _ in range(n)]
        self.filled = 0

    # places the symbol; raises on an illegal move
    def place(self, s, r, c):
        if not (0 <= r < self.n and 0 <= c < self.n):
            raise ValueError("Off board")
        if self.grid[r][c] != Symbol.EMPTY:
            raise ValueError("Cell taken")
        self.grid[r][c] = s
        self.filled += 1

    def is_full(self):
        return self.filled == self.n * self.n

    # only re-scan the row, column & diagonals touched by (r, c)
    def check_winner(self, s, r, c):
        n = self.n
        row = all(self.grid[r][i] == s for i in range(n))
        col = all(self.grid[i][c] == s for i in range(n))
        diag = r == c and all(self.grid[i][i] == s for i in range(n))
        anti = r + c == n - 1 and all(self.grid[i][n - 1 - i] == s for i in range(n))
        return row or col or diag or anti

class Game:
    def __init__(self, n, p1, p2):
        self.board = Board(n)
        self.players = [p1, p2]
        self.turn = 0
        self.status = GameStatus.IN_PROGRESS

    # one turn: validate, place, then test for win/draw
    def move(self, player, r, c):
        if self.status != GameStatus.IN_PROGRESS:
            raise RuntimeError("Game over")
        if player is not self.players[self.turn]:
            raise RuntimeError("Not your turn")

        self.board.place(player.symbol, r, c)
        if self.board.check_winner(player.symbol, r, c):
            self.status = GameStatus.WIN
        elif self.board.is_full():
            self.status = GameStatus.DRAW
        else:
            self.turn = (self.turn + 1) % 2   # alternate
        return self.status
Why this design scores well: checkWinner only inspects the line through the just-played cell, so it's O(N) per move instead of O(N²) for a full board scan. Modeling the cell as a Symbol enum (not a string or magic int) makes illegal states unrepresentable, and the explicit GameStatus plus turn index gives one unambiguous source of truth for "whose move is it and is the game over?"

Step 6 — Follow-up Extensions

Strong candidates anticipate where the interviewer pushes next:

  • O(1) win check: keep per-row, per-column, and two diagonal running counters. On each move increment the relevant counters (+1 for X, −1 for O); a magnitude of N means a win — no scanning at all.
  • Undo / replay: record each Move in a history list (Command/Memento) and pop to undo.
  • More players / bigger boards: generalize to K-in-a-row on an N×N board with M players.
  • AI opponent: plug in a MovePlayer strategy (random, minimax) behind the same interface.
  • Pluggable win rule: extract a WinStrategy so variants (Connect-style, gravity) drop in without touching Game.
Common mistakes: rescanning the entire board after every move (wasteful and signals you didn't think about cost); representing cells as raw strings or ints instead of an enum; forgetting to validate occupied/out-of-bounds cells; mutating the board before checking it's the right player's turn; and hardcoding 3×3 so the design can't generalize to N×N.
